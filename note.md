我理解你的需求。你想要实现一个类似于传统EDA工具中的原理图视图，能够清晰显示模块内部的信号连线关系，而不仅仅是模块块级的连接。让我梳理一下整体架构方案：

## 整体架构设计

### 1. 数据层重构

**当前问题**：
- 现有数据主要关注模块级别的连接
- 缺少模块内部信号级别的详细连接信息
- 端口信息只有静态描述，没有信号流向

**需要的数据结构**：
```
- 信号级连接图 (Signal-level netlist)
- 端口到信号的映射关系
- 信号的源和目标信息
- 信号位宽和类型信息
- 内部wire/reg信号定义
```

### 2. 视图层级架构

**三层视图结构**：

1. **系统级视图** (System Level)
   - 显示顶层模块间的连接
   - 类似当前的模块框图视图

2. **模块级原理图视图** (Module Schematic View) 
   - **这是你要的核心功能**
   - 显示选中模块内部的信号连线
   - 子模块以symbol形式展示
   - 信号线连接各个symbol的端口

3. **信号级详细视图** (Signal Detail View)
   - 显示具体信号的详细信息
   - 信号路径追踪

### 3. 原理图视图核心组件架构

**SchematicView 组件层次**：
```
SchematicView
├── SchematicCanvas (主画布)
├── ModuleSymbol (模块符号组件)
├── SignalWire (信号线组件)  
├── PortConnector (端口连接器)
├── SignalLabel (信号标签)
└── SchematicControls (视图控制)
```

### 4. 渲染策略

**Symbol-based 渲染**：
- 每个子模块渲染为标准化的矩形symbol
- Symbol显示模块名称和端口
- 端口按类型(input/output/inout)分布在symbol边缘

**信号连线渲染**：
- 使用SVG路径绘制信号线
- 支持正交路由(Manhattan routing)
- 信号线可显示位宽信息
- 支持总线表示(bus representation)

**布局算法**：
- 层次化布局(Hierarchical layout)
- 最小化信号线交叉
- 自动端口对齐

### 5. 交互设计

**多级导航**：
- 左侧层次树：选择要查看的模块
- 主视图：显示该模块的原理图
- 双击子模块：进入该子模块的内部原理图

**信号追踪**：
- 点击信号线高亮整个信号路径
- 显示信号的完整连接链
- 支持信号溯源和去向分析

### 6. 数据处理管道

**解析阶段**：
```
Verilog源码 → AST → 连接关系提取 → 信号级netlist
```

**处理阶段**：
```
Signal netlist → 布局计算 → 路由计算 → 渲染数据
```

**缓存策略**：
- 按模块缓存布局结果
- 信号连接关系预计算
- 视图状态持久化

### 7. 技术选型考虑

**渲染引擎**：
- 继续使用ReactFlow作为基础框架
- 或考虑切换到更适合原理图的库如Cytoscape.js
- 或直接使用SVG + D3.js构建自定义渲染器

**布局算法库**：
- dagre.js：层次化布局
- elk.js：更强大的图布局引擎
- 自定义原理图专用布局算法

### 8. 实现阶段规划

**Phase 1: 数据基础**
- 扩展Verilog解析器，提取信号级连接
- 构建信号netlist数据结构
- 实现基本的模块symbol渲染

**Phase 2: 原理图核心**
- 实现SchematicView组件
- 基本的信号线绘制
- 简单的自动布局

**Phase 3: 交互增强**
- 信号高亮和追踪
- 多级导航
- 布局优化算法

**Phase 4: 高级特性**
- 总线表示
- 参数化模块处理
- 性能优化

### 9. 挑战点

**技术挑战**：
- 复杂模块的自动布局质量
- 大规模设计的渲染性能
- 信号交叉最小化算法

**用户体验挑战**：
- 在有限屏幕空间内清晰显示复杂连接
- 多层次导航的流畅性
- 信息密度与可读性的平衡

这个架构将把你的工具从"模块框图查看器"升级为真正的"原理图查看器"，用户可以深入到任意模块内部查看详细的信号连接关系。
